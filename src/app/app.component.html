Welcome to @elemental-concept/env-bakery example app!

Quick Start

To start using @elemental-concept/env-bakery:

1. Install the library

$ npm i @elemental-concept/env-bakery

2. Create .env file in /assets

Make sure to add src/assets/.env to your .gitignore file! Environment files should never be commit to a repository!

To share base line environment file with your team members, create example.env which can then be renamed to .env and set up according to the needs of a specific environment. 

PRODUCTION=false

3. Convert your Angular environment into function

Open src/environments/environment.ts and convert environment from a static constant into a function and inject values from .env. 

import {{ '{' }} getEnv } from '@elemental-concept/env-bakery';

export const environment = () => (
  {{ '{' }}
    production: getEnv('PRODUCTION').boolean()
  }
);

4. Update main.ts to load environment before bootstrap

Please note that environment must be removed from default imports!

Environment must be loaded through a callback in main.ts. That does not affect components and services. 

import {{ '{' }} enableProdMode } from '@angular/core';
import {{ '{' }} platformBrowserDynamic } from '@angular/platform-browser-dynamic';
import {{ '{' }} bakeEnv } from '@elemental-concept/env-bakery';

import {{ '{' }} AppModule } from './app/app.module';

bakeEnv(() => import('./environments/environment')).then((environment: any) => {{ '{' }}
      if (environment.production) {{ '{' }}
    enableProdMode();
  }

  platformBrowserDynamic()
    .bootstrapModule(AppModule)
    .catch(err => console.error(err));
});

5. Remove environment.prod.ts

You don't need environment.prod.ts anymore when using @elemental-concept/env-bakery. Open angular.json and remove environment.prod.ts and environment.ts replacement. You can then remove environment.prod.ts from your project. 

"configurations": {{ '{' }}
  "production": {{ '{' }}
    "fileReplacements": []
  }
}

6. Don't forget to use environment as a function in your code

import {{ '{' }} Component } from '@angular/core';
import {{ '{' }} environment } from '../environments/environment';

@Component({{ '{' }}
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: [ './app.component.scss' ]
})
export class AppComponent {{ '{' }}
  readonly firebaseApiKey = environment().firebase.apiKey;

  constructor() {{ '{' }}
    console.log(`Firebase API key is "${{ '{' }} this.firebaseApiKey }".`);
  }
}

7. You can also use getEnv()

import {{ '{' }} Component } from '@angular/core';
import {{ '{' }} environment } from '../environments/environment';

@Component({{ '{' }}
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: [ './app.component.scss' ]
})
export class AppComponent {{ '{' }}
  readonly firebaseApiKey = getEnv('FIREBASE_API_KEY').string();

  constructor() {{ '{' }}
    console.log(`Firebase API key is "${{ '{' }} this.firebaseApiKey }".`);
  }
}

8. Set up your deployment system

When deploying your app make sure your deployment system puts a correct .env file. This depends on your deployment system. If you are using Docker, you can volume mount a file like this: 

docker run -v $(pwd)/.env:/root/app/dist/assets/.env my_docker_image
